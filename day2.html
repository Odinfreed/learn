🔍 Day 2 · 信息收集（主动扫描）

预计耗时：6-8小时 | 核心问题：在发动攻击前，如何尽可能多地了解目标？

阶段一：Nmap 高级扫描技术（预计1.5小时）

复习 Day 1 的扫描，深入学习 Nmap 高级参数，包括全端口扫描、服务版本深度探测、操作系统识别。

📌 任务

  • 对 Metasploitable 2 执行全端口扫描
  • 使用 -sV --version-intensity 9 深度探测服务版本
  • 使用 -O --osscan-guess 精准识别操作系统
  • 对比不同扫描类型（-sS, -sT, -sU）的差异
# 全端口扫描（65535个端口，速度适中）
nmap -p- -T4 靶机IP

# 服务版本深度探测
nmap -sV --version-intensity 9 靶机IP

# 操作系统精准识别
nmap -O --osscan-guess 靶机IP

# 组合命令（SYN扫描+版本+OS+全端口，耗时较长）
nmap -sS -sV -O -p- --osscan-guess 靶机IP

💡 思考问题1：SYN扫描（-sS）和TCP全连接扫描（-sT）有什么区别？为什么SYN扫描更隐蔽？

参考答案：
SYN扫描只发送SYN包，若收到SYN/ACK则回复RST断开，不建立完整连接，因此不会在目标系统留下连接日志，更隐蔽。
TCP全连接扫描会完成三次握手，建立完整连接，容易被日志记录。SYN扫描速度更快，但需要root权限。

阶段二：Nmap 脚本引擎（NSE）实战（预计2小时）

NSE 是 Nmap 最强大的功能，包含数百个安全检测脚本。学习使用默认脚本、分类脚本以及针对特定服务的脚本。

📌 任务

  • 使用默认安全脚本 -sC 扫描
  • 使用 SMB 相关脚本检测漏洞（如永恒之蓝）
  • 使用 HTTP 相关脚本枚举 Web 目录、标题、服务器信息
  • 尝试暴力破解（谨慎，仅限实验室）
# 默认安全脚本扫描
nmap -sC 靶机IP

# SMB 操作系统探测
nmap --script smb-os-discovery 靶机IP

# SMB 漏洞检测（包括 MS17-010）
nmap --script smb-vuln* 靶机IP

# HTTP 信息收集
nmap --script http-title,http-headers,http-enum 靶机IP

# FTP 匿名登录检测
nmap --script ftp-anon 靶机IP

# SSH 弱口令爆破（仅限实验室！）
nmap --script ssh-brute --script-args userdb=users.txt,passdb=pass.txt 靶机IP
💡 注意：暴力破解脚本可能触发目标系统的防御机制，仅在授权的实验室环境中使用。

💡 思考问题2：NSE 脚本的工作原理是什么？为什么某些脚本需要用户提供字典？

参考答案：
NSE 脚本使用 Lua 语言编写，通过发送特定数据包并分析响应来检测漏洞或收集信息。暴力破解类脚本需要字典是因为它们通过尝试大量用户名/密码组合来验证弱口令，字典质量直接影响成功率。

阶段三：扫描结果分析与资产报告（预计1.5小时）

将扫描结果整理成结构化报告，提取关键信息，为后续漏洞利用做准备。

📌 任务

  • 将扫描结果保存为多种格式（普通、grepable、XML）
  • 使用 grep、awk 等工具提取开放端口列表
  • 撰写资产报告，包含 IP、端口、服务版本、潜在风险
# 保存扫描结果为所有格式
nmap -sV -O -oA full_scan 靶机IP

# 从 grepable 文件中提取开放端口
grep "open" full_scan.gnmap | awk '{print $2,$3,$4,$5}' > open_ports.txt

# 使用 xsltproc 将 XML 转为 HTML 报告
xsltproc full_scan.xml -o report.html

💡 思考问题3：为什么资产报告需要包含服务版本信息？这对后续漏洞利用有何帮助？

参考答案：
服务版本信息可以快速匹配已知漏洞（如 vsftpd 2.3.4 对应后门漏洞）。精确的版本号使漏洞利用更加精准，避免浪费时间尝试不适配的 exploit。

阶段四：补充工具实战（预计1小时）

学习其他信息收集工具，拓宽视野，包括 masscan、nc、whatweb。

📌 任务

  • 使用 masscan 快速扫描常用端口
  • 使用 netcat 手动抓取 banner
  • 使用 whatweb 识别 Web 技术栈
# 安装 masscan
sudo apt install masscan -y

# 快速扫描1000个常用端口
masscan -p1-1000 --rate=1000 靶机IP

# 使用 netcat 抓取 FTP banner
nc -nv 靶机IP 21

# 使用 whatweb 识别 Web 技术
whatweb http://靶机IP

💡 思考问题4：masscan 与 nmap 相比有何优势？在什么场景下更适合使用 masscan？

参考答案：
masscan 以极高速率扫描（每秒数万包），适合大规模网络或快速探测开放端口。但缺乏深度服务识别，通常先用 masscan 发现开放端口，再用 nmap 进行详细扫描。

📌 任务

💡 思考问题5：被动信息收集与主动扫描的主要区别是什么？各有哪些优缺点？

参考答案：
主动扫描直接与目标交互，会留下日志，但信息精确；被动收集不接触目标，通过搜索引擎、历史数据等获取信息，隐蔽但可能不完整。通常两者结合使用。
