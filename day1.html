🔍 Day 2 · 信息收集（主动扫描）

预计耗时：6-8小时 | 核心问题：在发动攻击前，如何尽可能多地了解目标？

🎯 今日目标

  • 掌握 Nmap 高级扫描技术（全端口、服务识别、脚本扫描）
  • 学习操作系统识别、服务版本探测原理
  • 使用 Nmap NSE 脚本进行漏洞探测
  • 结合其他工具（如 masscan、nc）进行补充扫描
  • 编写一份规范的资产报告

⚙️ 实战步骤（6小时深度版）

阶段一：Nmap 基础回顾与高级选项（1.5小时）

复习 Day 1 的扫描命令，深入学习以下参数：

# 全端口扫描（65535个端口，速度适中）
nmap -p- -T4 靶机IP

# 服务版本深度探测
nmap -sV --version-intensity 9 靶机IP

# 操作系统精准识别
nmap -O --osscan-guess 靶机IP

# 同时进行端口、服务、操作系统扫描并输出所有格式
nmap -sS -sV -O -p- -oA full_scan 靶机IP

尝试不同的扫描类型：-sS (SYN半开)、-sT (TCP全连接)、-sU (UDP扫描)。注意 UDP 扫描耗时较长。

💡 思考：为什么 SYN 扫描更隐蔽？全端口扫描对目标网络的影响是什么？

阶段二：Nmap 脚本引擎（NSE）实战（2小时）

NSE 是 Nmap 最强大的功能，包含数百个安全检测脚本。

# 列出所有脚本分类
ls /usr/share/nmap/scripts/ | grep -v .nse

# 使用默认安全脚本
nmap -sC 靶机IP

# 针对特定服务使用脚本（如 SMB）
nmap --script smb-os-discovery 靶机IP
nmap --script smb-vuln* 靶机IP

# 使用 http 相关脚本
nmap --script http-title,http-headers,http-enum 靶机IP

# 尝试暴力破解（谨慎使用）
nmap --script ssh-brute --script-args userdb=users.txt,passdb=pass.txt 靶机IP

重点练习：

  • 使用 http-enum 枚举 Web 目录
  • 使用 smb-vuln-ms17-010 检测永恒之蓝漏洞
  • 使用 ftp-anon 检测匿名 FTP 登录
💡 深入思考：脚本扫描的结果是否完全可靠？如何避免误报？

阶段三：扫描结果分析与资产报告（1.5小时）

收集所有扫描信息，整理成结构化报告。

# 将扫描结果保存为 grepable 格式
nmap -oG output.gnmap 靶机IP

# 使用 awk 提取开放端口列表
grep "open" output.gnmap | awk '{print $2,$3,$4,$5}'

报告应包含：

  • 目标 IP、开放端口及服务版本
  • 操作系统猜测
  • 每个服务的潜在风险点（如 vsftpd 2.3.4 后门）
  • 截图关键扫描结果
💡 参考行业报告模板，尝试用 Markdown 或 Word 编写。

阶段四：补充工具实战（1小时）

学习使用其他信息收集工具，拓宽视野。

  • masscan：极速端口扫描器，适合大规模扫描
  • nc (netcat)：手动 Banner 抓取
  • whatweb：Web 技术栈识别
# masscan 快速扫描（1000个常用端口）
masscan -p1-1000 --rate=1000 靶机IP

# nc 抓取 banner
nc -nv 靶机IP 21

# whatweb 识别网站框架
whatweb http://靶机IP
💡 思考：masscan 与 nmap 的优缺点对比？什么场景下适合使用 masscan？
