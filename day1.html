📡 Day 1 · 渗透测试环境搭建

预计耗时：6-8小时 | 核心问题：如何搭建一个合法、可控的渗透测试实验室？

阶段一：虚拟化基础与VMware安装（预计1小时）

渗透测试需要隔离的环境，避免影响主操作系统。虚拟化技术让我们在一台物理机上同时运行多个独立系统。

📌 任务

  • 理解虚拟机网络模式（桥接、NAT、仅主机）
  • 下载并安装 VMware Workstation Pro（官方免费下载地址）
# 安装完成后，在Windows中检查VMware服务是否运行
services.msc
# 确保 VMware NAT Service 和 VMware DHCP Service 正在运行

💡 思考问题1：NAT模式和桥接模式的主要区别是什么？在渗透测试中，为什么通常使用NAT模式？

参考答案：
NAT模式：虚拟机通过宿主机共享IP访问外部网络，宿主机与虚拟机可以通信，但外部网络无法直接访问虚拟机。适合实验室环境，安全性高。
桥接模式：虚拟机与宿主机处于同一局域网，拥有独立IP，外部可直接访问。在渗透测试中，NAT模式更常用，因为它避免干扰外部网络，且便于与靶机隔离。

阶段二：部署Kali Linux攻击机（预计1.5小时）

Kali Linux是专为渗透测试设计的操作系统，内置600+工具。我们使用官方预置的虚拟机镜像，避免繁琐安装。

📌 任务

  • 下载Kali Linux VMware镜像：Kali官网下载页（选择VMware标签）
  • 解压并导入VMware，设置内存至少2GB，网络适配器NAT模式
  • 启动并登录（用户名kali，密码kali）
  • 设置root密码并更新系统
# 切换到root用户
sudo su
# 设置root密码
passwd root
# 更新软件源（可选，若网络慢可跳过）
apt update && apt upgrade -y
# 安装常用工具（可选）
apt install -y vim git

💡 思考问题2：为什么Kali Linux默认不推荐使用root用户？使用普通用户执行sudo有什么安全优势？

参考答案：
使用普通用户可以降低误操作风险，避免因权限过高导致系统损坏。sudo可以细粒度控制权限，并记录操作日志，提高安全性。

阶段三：部署Metasploitable 2靶机（预计1小时）

Metasploitable 2是一个故意留有大量漏洞的Linux虚拟机，专门用于练习渗透测试。

📌 任务

  • 下载Metasploitable 2：SourceForge下载页（约800MB）
  • 使用7-Zip解压（避免文件名损坏）
  • 在VMware中打开.vmx文件，设置网络适配器为NAT模式（与Kali一致）
  • 启动虚拟机，登录：用户名 msfadmin，密码 msfadmin
  • 查看IP地址：ifconfig
# 在Metasploitable中查看IP
ifconfig eth0
# 示例输出：inet addr:192.168.x.x

💡 思考问题3：Metasploitable 2故意保留大量漏洞，这反映了真实世界中的什么问题？

参考答案：
真实环境中很多系统因未及时更新、配置错误或使用过时软件而存在漏洞。Metasploitable模拟了这种“不安全的默认配置”，警示我们要及时打补丁、最小化服务、定期安全审计。

阶段四：网络连通性验证（预计30分钟）

确保Kali与Metasploitable可以互相ping通，这是后续一切攻击的基础。

📌 任务

  • 在Kali中ping靶机IP
  • 如果不通，检查网络适配器是否为同一NAT模式，重启VMware网络服务
# 在Kali终端中执行
ping 靶机IP -c 4
💡 如果ping不通，尝试：
1. 确认两台虚拟机网络适配器均为NAT模式。
2. 在Windows服务中重启VMware NAT Service和VMware DHCP Service。
3. 在Kali中执行 systemctl restart networking
4. 在Metasploitable中执行 sudo ifdown eth0 && sudo ifup eth0

💡 思考问题4：为什么在虚拟机中使用NAT模式就能实现互通，而桥接模式也可以？它们各有什么优缺点？

参考答案：
NAT模式下，虚拟机通过宿主机NAT设备通信，同一NAT网络内的虚拟机可以互通，且对外部网络不可见，安全。
桥接模式下，虚拟机直接接入物理网络，拥有独立IP，可以被外部直接访问，风险较高，但适合模拟真实服务器场景。

阶段五：首次扫描与侦察（预计1.5小时）

使用nmap对靶机进行扫描，发现开放端口和服务版本，这是信息收集的第一步。

📌 任务

  • 执行基础扫描
  • 执行全端口扫描（耗时较长）
  • 分析扫描结果，记录高危服务
# 基础扫描（端口、服务版本、操作系统）
nmap -sV -O 靶机IP

# 全端口扫描（65535个端口，约10-15分钟）
nmap -p- -T4 靶机IP

# 保存结果
nmap -sV -O -oA metasploitable_scan 靶机IP

预期结果应包含：21(FTP vsftpd 2.3.4), 22(SSH), 80(HTTP), 445(SMB), 3306(MySQL)等。

💡 思考问题5：从扫描结果中，你看到了哪些高危服务？这些服务可能带来什么风险？

参考答案：
vsftpd 2.3.4存在后门漏洞，可被直接利用获取shell；Samba版本较低，可能存在远程代码执行；MySQL默认root空密码可被利用；Apache 2.2.8也存在多个已知漏洞。这些都是渗透测试的突破点。

阶段六：环境备份与法律合规（预计30分钟）

创建虚拟机快照以便随时恢复初始状态，并学习渗透测试的法律边界。

📌 任务

  • 在VMware中为Kali和Metasploitable分别创建快照（关闭虚拟机后操作）
  • 阅读相关法律知识：Offensive Security 法律声明

💡 思考问题6：为什么在开始渗透测试前必须获得授权？未经授权的渗透测试可能带来什么后果？

参考答案：
未经授权扫描或攻击他人系统属于违法行为，可能触犯《网络安全法》《刑法》等，面临法律责任。合法渗透测试必须签订授权协议，明确测试范围和边界。
