📡 Day 1 · 渗透测试环境搭建

预计耗时：6-8小时 | 核心问题：如何搭建一个合法、可控的渗透测试实验室？

🎯 今日目标

  • 理解虚拟化原理，安装 VMware Workstation Pro
  • 下载并配置 Kali Linux 攻击机（虚拟机）
  • 下载并配置 Metasploitable 2 靶机（虚拟机）
  • 掌握虚拟机网络模式（NAT/桥接/仅主机）并实现互通
  • 使用 nmap 进行首次扫描，验证环境
  • 了解渗透测试流程与法律边界

⚙️ 实战步骤（6小时深度版）

阶段一：虚拟化基础（1小时）

理论： 什么是虚拟化？为什么渗透测试需要隔离环境？

  • 学习 VMware 三种网络模式：桥接、NAT、仅主机
  • 下载 VMware Workstation Pro（官方免费下载）
  • 以管理员身份安装，重启电脑
# 验证 VMware 服务是否运行（Windows）
services.msc
# 确保 VMware NAT Service 和 VMware DHCP Service 正在运行

阶段二：部署 Kali Linux（1.5小时）

下载 Kali Linux 官方虚拟机镜像：Kali VMware 镜像（约3.5GB）

  • 解压 .7z 文件（使用 7-Zip）
  • 在 VMware 中打开 .vmx 文件
  • 启动虚拟机，登录：kali / kali
  • 设置 root 密码并更新系统
sudo passwd root          # 设置 root 密码
sudo su                   # 切换到 root
apt update && apt upgrade -y   # 更新系统（耗时较长，可后台运行）
apt install -y kali-linux-headless   # 安装核心工具集（可选）
💡 更新系统可能需要30-60分钟，建议利用这段时间学习《Kali Linux 工具集概览》或继续下一步。

阶段三：部署 Metasploitable 2（1小时）

下载 Metasploitable 2：SourceForge 链接（约800MB）

  • 使用 7-Zip 解压（避免文件名损坏）
  • 在 VMware 中打开 .vmx 文件，设置网络适配器为 NAT模式
  • 启动虚拟机，登录：msfadmin / msfadmin
  • 查看 IP 地址：ifconfig
# 在 Metasploitable 中执行
ifconfig eth0
# 记录 IP（例如 192.168.xxx.xxx）

阶段四：网络连通性验证（1小时）

在 Kali 中 ping 靶机 IP，确保网络互通。

ping 靶机IP -c 4

如果不通，排查步骤：

  • 确认两台虚拟机网络适配器均为 NAT
  • 检查 VMware NAT 服务是否运行
  • 在 Kali 中重启网络：systemctl restart networking
  • 在靶机中重启网络：sudo ifdown eth0 && sudo ifup eth0

阶段五：首次扫描与侦察（1.5小时）

使用 nmap 对靶机进行基础扫描，理解开放端口背后的服务。

# 基础扫描
nmap -sV -O 靶机IP

# 全端口扫描（耗时约10分钟）
nmap -p- -T4 靶机IP

# 保存结果
nmap -sV -O -oA metasploitable_scan 靶机IP

分析扫描结果：

  • 开放了哪些端口？21(FTP), 22(SSH), 80(HTTP), 445(SMB), 3306(MySQL) 等
  • 服务版本 vsftpd 2.3.4 存在已知后门，后续将利用
  • 操作系统为 Linux 2.6.x
💡 深入思考：为什么 Metasploitable 故意保留这么多漏洞？这反映了真实世界的哪些安全问题？

阶段六：环境加固与快照（30分钟）

在 VMware 中为两台虚拟机创建快照，方便后续随时恢复初始状态。

  • 关闭虚拟机 → 右键 → 快照 → 拍摄快照
  • 命名：Kali_initial, Metasploitable_initial

学习如何利用快照回滚，避免重复安装。
